# План улучшения проекта uz++ (WORK_PLAN.md)

В ходе анализа репозитория были найдены заглушки (stubs) и недоделанные файлы (со статусом PARTIAL и EXPERIMENTAL). Ниже представлен пошаговый план по их улучшению без удаления существующего рабочего кода.

## Шаг 1: Улучшение инструментов разработки (LSP и DAP серверы)
В исходном коде серверов присутствуют явные комментарии `// Implementation of missing LSP methods` и `// Implementation of missing DAP methods`. Эти компоненты критически важны для удобства разработчиков в VS Code.
- **`src/lsp_server.cpp`**: Реализовать или улучшить `Signature Help` (сейчас возвращает пустую заглушку), более точное извлечение текста при изменениях.
- **`src/dap_server.cpp`**: Доработать команды отладчика для полноценного отображения переменных.

## Шаг 2: Доработка частично реализованных модулей Stdlib (PARTIAL -> REAL)
В `docs/stdlib-status.md` указаны модули, функционал которых реализован не до конца:
- **`stdlib/tarjima.hpp`**: Реализовать поддержку парсинга JSON для файлов локализации.
- **`stdlib/xatoliklar.hpp`**: Заменить заглушки обработчиков сигналов (signal handlers) на рабочую реализацию.
- **`stdlib/kesh.hpp`**: Дописать логику кэширования в памяти (замена заглушек).
- **`stdlib/veb_ui.hpp`**: Завершить Redux-шаблоны и улучшить HTML/HTMX генерацию.

## Шаг 3: Перевод экспериментальных модулей в рабочее состояние (EXPERIMENTAL -> PARTIAL/REAL)
- **`stdlib/grafika/reaktiv.hpp`** и **`stdlib/grafika/hodisalar.hpp`**: Связать структуры событий и паттерн наблюдателя (Observer) с реальным рендерингом (из `oyna.hpp`).
- **`stdlib/suniy_intellekt.hpp`**: Доработать вызовы к Ollama LLM, добавить fallback для случаев, когда curl недоступен.
- **`stdlib/platforma.hpp`**: Наполнить макросы определения платформы реальным функционалом для API.
